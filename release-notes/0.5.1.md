# 0.5.1

A UI + reliability release. (0.5.0 was never published separately, so this rolls up its changes too — if you're coming from 0.4.0 you get everything below.)

## Reinstall step

The focus-stealing fix and the daemon version bump are in the host service. After updating the desktop app, open each server's **Management Service** card → **Install / Update** to push the refreshed host service. The UI changes take effect as soon as the desktop app updates.

## New

- **Left-rail server navigation.** The server sub-tabs (Dashboard, Update, Pods, Users, Admin, Welcome Package, Automated tasks) now run down the left side under the server name.
- **Distinct OLED theme.** The OLED theme is no longer just darker surfaces — it now has its own electric-cyan accent, cool-white text, and brighter status colors on true black, so it reads as a genuinely different interface.
- **Refreshed app icon** — a new gold-diamond mark. (On an in-place update, Windows may show the old cached icon until the icon cache refreshes or you reboot.)
- **Grant Specialization XP** for a player's tracks (Combat / Crafting / Gathering / Exploration / Sabotage), as a guarded offline database write.
- **Live Director (BGD) status** — accurate per-partition startup→healthy phases, true per-pod uptime, and live online / grace-period / transit / offline player states on the Users tab.
- **Customizable restart schedule** — a full cron expression (e.g. weekdays-only) in addition to the simple daily time.

## Fixed

- **No more console-window flashes / focus stealing on Windows.** Background commands the manager runs (PowerShell / VM status checks) could briefly flash a console window or steal focus from a fullscreen game. Spawned processes now redirect stdin and run PowerShell hidden + non-interactive, and the server detail view skips redundant VM status polling while the server is healthy.
- **Hyper-V Start / Stop VM controls work again.** The VM is now resolved by host-side detection, so power controls work even though the in-cluster world name differs from the Hyper-V VM name (previously "VM not found" + disabled buttons).
- **Form controls no longer clipped in a non-maximized window.** The content area is now a proper scroll container at any window size, so tall forms (e.g. Admin) and their buttons stay reachable.
- **Real-time backup-failure diagnostics** — the dump pod's logs / exit reason are captured live, plus a stray-table ownership precheck, instead of an opaque "backup failed."

## Changed

- **Online players are blocked from currency / Intel / Specialization-XP grants up front** (disabled Publish + a clear note, plus a pre-submit re-check), since the game overwrites these database edits when a player logs in.

## Maintenance

- Dependabot now watches npm, Cargo, and GitHub Actions; CI gained a RustSec `cargo audit` job; Vite upgraded (dev-tooling security advisories cleared); CI actions moved to Node 24.

## Notes

- No configuration changes are required. Existing schedules, welcome packages, grants, and enable switches keep their settings.
