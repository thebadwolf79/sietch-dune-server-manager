# 0.5.2

A UI + reliability release that leans the interface into the Funcom Dune: Awakening launcher's look. (0.5.0 and 0.5.1 were never published separately, so this rolls up their changes too — coming from 0.4.0 you get everything below.)

## Reinstall step

The focus-stealing fix and the daemon version bump live in the host service. After updating the desktop app, open each server's **Management Service** card → **Install / Update** to push the refreshed host service. UI changes apply as soon as the desktop app updates.

## Look & feel — Funcom launcher motif

- **Gold metallic "SIETCH" wordmark** for branding, echoing the launcher's gold Dune wordmark.
- **Electric-cyan for active / selected / focus states** (left-rail active tab, focus rings) while warm gold/bronze stays the brand accent — the launcher's gold-brand + cyan-interactive split.
- **Left-rail server navigation** with angular, beveled panel framing.
- **Distinct OLED theme** — its own electric-cyan accent, cool-white text, and brighter status colors on true black, so it reads as a different interface rather than just darker.
- **Refreshed gold-diamond app icon.** (On an in-place update, Windows may show the old cached icon until the icon cache refreshes or you reboot.)

## New

- **Grant Specialization XP** per track (Combat / Crafting / Gathering / Exploration / Sabotage) — a guarded offline database write.
- **Live Director (BGD) status** — accurate per-partition startup→healthy phases, true per-pod uptime, and live online / grace-period / transit / offline player states on the Users tab.
- **Customizable restart schedule** — a full cron expression (e.g. weekdays-only) in addition to the simple daily time.

## Fixed

- **No more console-window flashes / focus stealing on Windows.** Background commands now redirect stdin and run PowerShell hidden + non-interactive, and the detail view skips redundant VM status polling while the server is healthy — so nothing flashes a console or steals focus from a fullscreen game.
- **Admin "Publish" button is now obvious.** It's left-aligned (no more hunting across a wide window or finding it clipped when not maximized) and a solid, bright, angular call-to-action instead of a muted ghost button that blended into the panel.
- **Hyper-V Start / Stop VM controls work again** — the VM is resolved by host-side detection, so power controls work even though the in-cluster world name differs from the Hyper-V VM name.
- **Form controls no longer clipped in a non-maximized window** — the content area is a proper scroll container at any size.
- **Real-time backup-failure diagnostics** — captures the dump pod's logs / exit reason live, plus a stray-table ownership precheck.

## Changed

- **Online players are blocked from currency / Intel / Specialization-XP grants up front** (disabled Publish + a clear note + a pre-submit re-check), since the game overwrites these edits on login.

## Maintenance

- Dependabot now watches npm, Cargo, and GitHub Actions; CI gained a RustSec `cargo audit` job; Vite upgraded (dev-tooling advisories cleared); CI actions moved to Node 24.

## Notes

- No configuration changes are required. Existing schedules, welcome packages, grants, and enable switches keep their settings.
