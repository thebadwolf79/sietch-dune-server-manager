# 0.5.0

A UI and quality-of-life release: the server view gets a left-rail navigation and a refreshed app icon, the Admin grants now stop you up front when a player is online, the Hyper-V Start/Stop VM controls work again, and a layout bug that hid form buttons in a non-maximized window is fixed.

This builds on everything in 0.4.0 (specialization-XP grants, live Director/BGD status + grace-period/transit player tracking, real-time backup-failure diagnostics, and the cron-based restart schedule).

## Reinstall step

All of 0.5.0's changes are in the desktop app, so they take effect as soon as you update it. The bundled host service is version-bumped to 0.5.0 only — pushing it via each server's **Management Service** card → **Install / Update** is optional (it just keeps the reported version in sync; no behavior changed in the service itself).

## New

- **Left-rail server navigation.** The server sub-tabs (Dashboard, Update, Pods, Users, Admin, Welcome Package, Automated tasks) now run down the left side under the server name instead of across the top, matching the redesigned look.
- **Refreshed app icon** — a new gold-diamond mark.

## Fixed

- **Hyper-V Start / Stop VM controls work again.** The dashboard addressed the VM by its in-cluster world/battlegroup id (e.g. `sh-…`), which is not the VM's Hyper-V name, so the lookup failed with "VM not found" and the Start/Stop buttons were disabled. The app now resolves the real VM by host-side detection (falling back to it whenever the requested name isn't an exact match), so state reads and Start/Stop work even though the names differ.
- **Form controls no longer get clipped in a non-maximized window.** In a narrower window, parts of the Admin form (the value field and the publish button) could be pushed off-screen with no way to scroll to them. The content area is now a proper scroll container at any window size.

## Changed

- **Online players are blocked from currency / Intel / Specialization-XP grants up front.** Because the game overwrites these database edits when a player logs in, the grant must be applied while the player is offline. The Admin form now checks the selected player's live status, disables **Publish** with a clear note when they're online ("must be logged out to the main menu"), and re-checks immediately before sending so a login after selection is still caught. The service already refused online grants; this makes it obvious before you submit.

## Notes

- No configuration changes are required. Existing schedules, welcome packages, grants, and enable switches keep their current settings.
